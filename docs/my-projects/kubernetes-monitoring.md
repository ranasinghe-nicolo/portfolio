# Kubernetes Monitoring

Progetto di osservabilità per un cluster Kubernetes con metriche, dashboard e alerting.

## Obiettivo

Migliorare visibilità e resilienza del cluster attraverso monitoraggio end-to-end.

## Attività principali

- deployment di Prometheus e Grafana
- raccolta metriche da cluster e applicazioni
- creazione di dashboard per utilizzo CPU, memoria e latenza
- configurazione di alert per errori e soglie critiche

## Risultati

- tempi di individuazione dei problemi ridotti
- visibilità completa dei carichi applicativi
- base per analisi delle prestazioni nel tempo
